50
513. To register properties: include your property name and variable pointer in .property()
52 > Name will show up exactly as-is in the Inspector
53 > In place of putting in a variable, you can put in function poitners for getter/setter functions too.
54 > This is useful if everytime the value gets modified, you want some additional logic to run
55
564. To make properties read-only, use .property_readonly() instead.
57 > If using getter/setter functions to register, setter functions are not needed.
58
595. To add metadata, follow the format above to add more metadata. Also tell Noah to make use of this metadata.
60 > There can be multiple metadata per property. Add more by adding a comma. e.g. metadata(xxx),metadata(xxx),metadata(xxx)
61 > Make sure the last metadata does not have a trailing comma.
62
636. Do note that the ORDER of registration will be the order that shows up in the inspector.
